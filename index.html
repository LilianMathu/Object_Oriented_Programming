Concepts of object oriented programming

Modules

A module has its own namespace at the module level. They do not pollute the global space.

Classes

A class is a blueprint. Classes are created using the class keyword followed by a className whose first letter is capitalized.

In ES6, a class is a type of a function.

The output of #console.log(typeOf className)# will give you a function.

In ES5, there were no classes. Classes in ES6 acts like constructor functions in ES6.

Constructors

To initialize information for a class instance (object) we do it in a constructor using the constructor keyword. Arguments in a constructor are used to initiate objects. You can pass arguments to the constructor like shown below:
Class Drone{ constructor(id, Name){ console.log(id +" " +name); } } let drone = new Drone(1234 "Cool Drone");

Best practice is to reduce the operations in a constructor, only include the instance variables. Instance variables are created in the constructor function in the class. The this keyword is used to refer to the instance being created.

Objects

An onject is an instance of a class. To create an object, the new keyword is used.
let drone = new Drone.
if you log out typeof drone, which is the name of the new object, the result is an object.


After instanciating the variables with the this keyword in the constructor function, you can access them using the dot notation or the bracket notation. In bracket notation properties are accessed as strings.

console.log("drone: " +drone.id +" " +drone.name); //dot notation.
console.log("drone: " +drone['id'] +" " +drone['name']); //bracket notation.


Instance variables (Properties) and Statis (class) properties

Instance variables are properties of a specific instance of the variable. Consider the code below

let drone = new Drone(1234, "cool drone");

let drone1 = new Drone(4567, "awesome drone");

The values of drone and drone1 are instance variables. Each instance will have different instance variables.

However, we can have properties that are common to all drones, in addition to their instance properties. These properties are referred to as the class or static properties.
Static properties are added after the closing class bracket.


Methods

In addition to having properties on classes, we can also add methods. A method is a function that gets attached to the instance.

fly(){
console.log("Drone: " +this.id +" " +"is flying");
}


->fly is a method attached to each instance. After creating the methods in the class body, you need to call them in an instance.
fly.drone();


NB: It is important to note that properties of an instance are accessed using the this keyword.

Static methods

The static keyword defines a static method for a class. Static methods aren't called on instances of the class. Instead, they're called on the class itself. These are often utility functions, such as functions to create or clone objects.

class Drone{
static getCompany(){
console.log("in my company");
}
}
Drone.getCompany();


Getters and setters

Getters and setters can be used in a class.
Private properties have an underscore character before the property name. For example this._id = id. Getters and seters are used to create properties that look like functions.
